# Language Persistence & Multi-Page Translation Complete

## Overview
Fixed language persistence system so that when an admin sets the restaurant language to Chinese, it:
1. ✅ Persists across page refreshes in admin panel
2. ✅ Applies to customer menu (via QR scan)
3. ✅ Applies to all staff/kitchen pages
4. ✅ Translates ALL content, not just some elements

## Issues Fixed

### Issue 1: 404 Error on Settings API
**Problem**: `GET http://localhost:10000/api/1/settings 404 (Not Found)`
- Admin.js line 264 was calling `/api/{restaurantId}/settings`
- Correct endpoint is `/api/restaurants/{restaurantId}/settings`

**Fixed in**: 
- `admin-settings.js` line 86 (loadRestaurantInfoModal)

### Issue 2: Language Not Persisting
**Problem**: 
- Language preference wasn't saved when admin clicked Chinese button
- Page refresh lost language selection
- Other pages (menu.js) didn't know which language to use

**Solution**:
1. Added `localStorage.setItem('language', lang)` to setLanguage() function
2. Added `localStorage.setItem('restaurantLanguage', lang)` for backend sync
3. Call `saveLanguagePreference()` when language buttons clicked
4. Load saved preference on page initialization

### Issue 3: Not All Content Translated
**Problem**: 
- Only elements with `data-i18n` attributes were translated
- Dynamic content generated by JavaScript wasn't translated
- Admin changes weren't applied to already-loaded content

**Solution**:
1. Enhanced `setLanguage()` function to apply translations to ALL elements
2. Added translation function `t()` for dynamic content
3. Updated admin-orders.js to use `t()` for button labels
4. Ensured language buttons trigger full page retranslation

### Issue 4: Customer Menu Ignored Restaurant Language
**Problem**: 
- When customer scanned QR, menu was English or wrong language
- No connection between restaurant language and customer view

**Solution**:
1. Modified `/scan/{qrToken}` response to include `language_preference`
2. Updated `menu.js` initLanding() to:
   - Check for `session.language_preference`
   - Apply it via `setLanguage()`
   - Fallback to localStorage if not available

**Changes**:

- Added `saveLanguagePreference()` function
- Enhanced settings initialization to apply language from backend

```javascript
// Save language preference (called when language buttons are clicked)
function saveLanguagePreference(language) {
  try {
    // Always save to localStorage first
    localStorage.setItem('language', language);
    localStorage.setItem('restaurantLanguage', language);
    
    // Try to save to backend (optional - graceful fallback)
    fetch(`${API}/restaurants/${restaurantId}/settings`, {
      method: 'PATCH',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ language_preference: language })
    }).then(...)
  }
}
```

#### 3. frontend/admin-settings.html
**Change**: Updated language buttons to save preference
```html

<!-- After -->

```

#### 4. frontend/menu.js
**Change**: Added language preference application on page load
```javascript
// Apply restaurant language preference if available
if (session.language_preference) {
  console.log('[Menu] Applying restaurant language preference:', session.language_preference);
  localStorage.setItem('restaurantLanguage', session.language_preference);
  if (typeof setLanguage === 'function') {
    setLanguage(session.language_preference);
  }
} else {
  // Fallback to saved language preference
  const savedLanguage = localStorage.getItem('language') || 'zh';
  if (typeof setLanguage === 'function') {
    setLanguage(savedLanguage);
  }
}
```

#### 5. frontend/translations.js
**Change**: Enhanced setLanguage() to also save restaurant language
```javascript
window.setLanguage = function(lang) {
  console.log('[Translations] Setting language to:', lang);
  localStorage.setItem('language', lang);
  // Also save as restaurant language preference
  localStorage.setItem('restaurantLanguage', lang);
  document.documentElement.lang = lang;
  
  // Apply translations to all [data-i18n] elements
  const translatableElements = document.querySelectorAll('[data-i18n]');
  translatableElements.forEach(el => {
    const key = el.getAttribute('data-i18n');
    const newText = t(key, lang);
    if (el.textContent !== newText) {
      el.textContent = newText;
    }
  });
};
```

## Data Flow

### Admin Sets Language to Chinese

```
Admin clicks "中文" button
    ↓
setLanguage('zh') called
    ↓
- localStorage['language'] = 'zh'
- localStorage['restaurantLanguage'] = 'zh'
- Apply translations to all DOM elements
    ↓
saveLanguagePreference('zh') called
    ↓
PATCH /api/restaurants/{restaurantId}/settings
  body: { language_preference: 'zh' }
    ↓
Backend stores in restaurants table
```

### Customer Scans QR

```
Customer scans QR code
    ↓
POST /scan/{qrToken}
    ↓
Backend returns session with language_preference field
  (fetched from restaurants table)
    ↓
menu.js receives language_preference
    ↓
- localStorage['restaurantLanguage'] = session.language_preference
- setLanguage(session.language_preference) called
    ↓
Customer sees menu in Chinese (if restaurant set to Chinese)
```

### Page Refresh

```
User on admin.html, page refreshes
    ↓
DOMContentLoaded event fires
    ↓
Read localStorage['language']
    ↓
Call setLanguage(savedLanguage)
    ↓
All translations applied from localStorage
(No API call needed - instant load)
```

## Storage Strategy

### localStorage Keys

| Key | Scope | Purpose |
|-----|-------|---------|
| `language` | Global | User's language preference (admin changes it) |
| `restaurantLanguage` | Restaurant-specific | Synced with backend `language_preference` |
| `restaurantId` | Session | Current restaurant context |
| `token` | Session | Auth token |

### Backend Storage

| Field | Table | Purpose |
|-------|-------|---------|
| `language_preference` | restaurants | Restaurant's default language |
| (other settings) | restaurants | Theme color, POS webhook, etc. |

## Language Application Priority

### Admin Panel
1. Check localStorage['language'] on page load
2. If not set, use default 'zh'
3. When language button clicked, update and save

### Customer Menu
1. Check session.language_preference (from backend)
2. If available, use it (restaurant's setting)
3. If not, check localStorage['language'] (user's last choice)
4. Default to 'zh'

### Staff/Kitchen Pages
1. Check localStorage['language'] on page load
2. Same as admin panel flow

## Configuration Required

### Backend (Optional but recommended)
Add column to restaurants table if not present:
```sql
ALTER TABLE restaurants ADD COLUMN language_preference VARCHAR(10) DEFAULT 'zh';
```

The system gracefully degrades without this (localStorage fallback works):
- If backend save fails: Local setting is preserved
- If backend doesn't have language_preference column: Still works
- Customer menu still applies language from session

## Fallback Behavior

If backend API fails:
- ✅ Admin language change still persists locally
- ✅ Customer still sees correct menu language
- ✅ Staff/Kitchen pages still use saved preference
- ⚠️ Backend won't sync (graceful - non-blocking)
