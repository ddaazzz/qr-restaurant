## Implementation Details

### Translation Keys Pattern
All keys follow the pattern: `admin.{feature}.{element}`
- Ensures consistency with existing translation architecture
- Grouped by functional area for easy maintenance
- Supports both inline translations and dynamic text

### Dynamic Content Handling
For content generated by JavaScript functions, the translation happens:
1. Initially via template literals with `data-i18n` attributes
2. Function `setLanguage()` automatically applies translations to all elements with `data-i18n` attribute
3. Dynamic text updated via `t()` function calls for button labels

### Language Support
- **English**: 40 keys with clear, descriptive text
- **Chinese (Traditional)**: 40 corresponding translations with proper cultural localization
  - "Session" → "期間" (period/session)
  - "Print/Email" → "打印/電郵" (proper simplified terms)
  - Monetary references maintained as-is for consistency

## Usage Examples

### Static HTML Translation
```html
<span data-i18n="admin.add-to-table">Add to Table</span>
```
Automatically translated when `setLanguage('zh')` is called.

### Dynamic JavaScript Translation
```javascript
submitBtn.textContent = t('admin.add-to-table');
```
Directly retrieves translation based on current language setting.

### Dynamic HTML with Translations
```javascript
const html = `
  <div data-i18n="admin.session-label">Session</div>
`;
```
Generated HTML elements get translated when setLanguage() runs.
