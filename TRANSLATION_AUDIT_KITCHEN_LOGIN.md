## Implementation Notes

1. **HTML Element Attributes:**
   - Elements using `data-i18n` for text content are already set up for translation
   - Elements using `data-i18n-placeholder` for placeholder text are already configured
   - New elements need these attributes added

2. **JavaScript-Generated Content:**
   - Content rendered from `kitchen.js` needs to reference translation keys
   - Status values ("pending", "preparing", "served") should be passed through translation function
   - Order type formatting should use translation keys
   - Time format strings need translation

3. **Files Affected for Implementation:**
   - `frontend/translations.js` - Add new translation keys
   - `frontend/kitchen.html` - Add `data-i18n` attributes to "Language" label
   - `frontend/kitchen.js` - Update renderKitchenOrders, updateAllItemStatus, getTimeAgo functions to use translation keys
   - `frontend/login.html` - Already mostly translated
   - `frontend/login.js` - Update error message handling to use translation keys

4. **Priority:**
   - HIGH: kitchen.html button labels and status messages (user-facing on kitchen display)
   - HIGH: Time format translations (displayed for every order)
   - MEDIUM: Alert messages
   - LOW: Page titles and meta tags
